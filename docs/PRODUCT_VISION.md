# GDELT MCP 产品愿景

## 现状 vs 目标

| 维度 | 现状（数据查询工具） | 目标（事件洞察助手） |
|------|-------------------|-------------------|
| **用户请求** | "查询华盛顿1月的冲突事件" | "华盛顿最近发生了什么大事？" |
| **系统回复** | 原始数据表格 | 洞察摘要 + 关键事件时间线 |
| **交互方式** | CLI + 技术参数 | Web + 自然语言 |
| **响应时间** | 60秒（实时聚合） | <5秒（预计算） |
| **事件引用** | GlobalEventID 742447353 | US-20240115-WDC-PROTEST-001 |
| **核心能力** | 数据检索 | 因果分析 + 态势感知 |

## 核心用户画像

### 用户1: 国际新闻分析师
- **痛点**: 需要快速了解某地区近期局势
- **场景**: "给我讲讲红海危机的发展"
- **需求**: 事件时间线、相关方、影响评估

### 用户2: 学术研究员
- **痛点**: 需要追踪特定主题的历史演变
- **场景**: "分析2024年以色列冲突对油价的影响"
- **需求**: 数据导出、多维度筛选、长期趋势

### 用户3: 情报官员
- **痛点**: 需要发现异常模式
- **场景**: "检测哪些地区冲突强度异常上升"
- **需求**: 异常检测、实时监控、预警推送

## 技术演进路径

```
Phase 1: 产品重塑 (2周)
├── 定义用户场景
├── 设计产品架构
└── 输出PRD文档

Phase 2: 数据优化 (2周)  
├── 预计算表
├── ETL Pipeline
└── 事件指纹系统

Phase 3: 工具重构 (2周)
├── 精简工具集 (15→5)
├── 意图识别优化
└── 高频查询缓存

Phase 4: 前端构建 (4周)
├── React + TypeScript
├── Leaflet地图
└── ECharts图表

Phase 5: 智能增强 (2周)
├── LLM摘要生成
├── 因果链分析
└── 异常检测

总计: 12周 (3个月)
```

## 风险评估

| 风险 | 影响 | 缓解措施 |
|-----|------|---------|
| 预计算ETL太慢 | 高 | 增量更新、并行处理 |
| LLM成本过高 | 中 | 本地小模型、按需调用 |
| 前端开发延期 | 中 | 分阶段交付、MVP优先 |
| 数据质量问题 | 高 | 数据清洗Pipeline |
