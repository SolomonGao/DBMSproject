## 核心改进点

### 1. 预计算架构

**问题**: 实时GROUP BY 15M行数据太慢

**解决**: 预计算 + ETL Pipeline

```
Daily ETL (凌晨2点)
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  Raw Events │────→│  Transform  │────→│ Pre-computed│
│  (15M)      │     │  (聚合/指纹) │     │ Tables      │
└─────────────┘     └─────────────┘     └─────────────┘
                                              │
                    ┌─────────────┐           │
User Query ────────→│  Cache Hit  │←──────────┘
                    │  (< 1s)     │
                    └─────────────┘
```

### 2. 意图驱动工具

**V1 (参数驱动)**:
```python
query_by_actor(actor="USA", date_start="2024-01-01", 
               date_end="2024-01-31", event_type="conflict")
```

**V2 (意图驱动)**:
```python
search_events(query="1月华盛顿的抗议")
# 系统自动解析: time=2024-01, location=Washington, type=protest
```

### 3. 事件指纹系统

**可读ID**:
```
旧: GlobalEventID 742447353 (无意义)
新: US-20240115-WDC-PROTEST-001 (可读)
     │    │      │    │      │
     │    │      │    │      └── 序号
     │    │      │    └───────── 事件类型
     │    │      └────────────── 地点缩写
     │    └───────────────────── 日期
     └────────────────────────── 国家代码
```

### 4. LLM增强

**摘要生成**:
```
输入: Actor1=USA, Actor2=Iran, Event=Conflict, Location=Persian Gulf
输出: "美伊波斯湾军事对峙"
```

**因果分析**:
```
Event A (01-12) ──→ Event B (01-14) ──→ Event C (01-15)
(美宣布制裁)       (伊扣押商船)         (美派航母)
     │                   │                   │
     └───────────────────┴───────────────────┘
              Causal Chain
```

## 技术栈

| 层级 | 技术 | 说明 |
|-----|------|------|
| 前端 | React + TypeScript + Leaflet | 地图 + 图表 |
| MCP Client | Python + mcp-sdk | 协议实现 |
| Router | Ollama + Qwen 2.5B 3B | 本地推理 |
| LLM | Kimi Code API | 主要推理 |
| MCP Server | FastMCP | 工具注册 |
| 数据库 | MySQL 8.0 + 预计算表 | 15M + 索引 |
| ETL | Python + aiomysql | 异步Pipeline |
| 缓存 | In-Memory LRU | 300s TTL |
